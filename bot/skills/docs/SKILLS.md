# Skills Framework

本框架採用三層架構設計，仿照 OpenClaw 的設計理念。

## 架構概覽

```
┌─────────────────────────────────────────────────┐
│                  Permissions                     │
│         (誰可以使用什麼技能)                      │
├─────────────────────────────────────────────────┤
│                    Skills                        │
│         (組合工具完成任務)                        │
├─────────────────────────────────────────────────┤
│                    Tools                         │
│         (底層 API 操作)                          │
└─────────────────────────────────────────────────┘
```

## 三層說明

### 1. Tools (工具鏈)
- 最底層，純粹的 API 操作
- 無業務邏輯，只負責執行單一功能
- 例如：`google_calendar_api.list_events()`

### 2. Skills (技能鏈)
- 中間層，組合多個 tools 完成任務
- 包含業務邏輯和錯誤處理
- 例如：`CalendarSkill.get_today_schedule()`

### 3. Permissions (權限鏈)
- 最上層，控制存取權限
- 定義誰可以使用什麼技能
- 支援用戶級別、群組級別權限

## 目錄結構

```
bot/
├── core/
│   ├── __init__.py
│   ├── base_tool.py       # 工具基類
│   ├── base_skill.py      # 技能基類
│   └── permissions.py     # 權限管理
├── skills/
│   ├── docs/
│   │   └── SKILLS.md      # 本文件
│   └── calendar/
│       ├── SKILL.md       # 技能定義文檔
│       ├── __init__.py
│       ├── tools.py       # 工具層
│       └── skill.py       # 技能層
```

## 新增技能流程

1. **編寫 SKILL.md** - 定義技能的功能、輸入輸出、權限需求
2. **實作 tools.py** - 實現底層 API 操作
3. **實作 skill.py** - 組合工具，實現業務邏輯
4. **註冊權限** - 在 permissions 中註冊技能權限
